Below is the advanced Replit Agent prompt for the mobile app version of UTA FurniFind. It keeps all the same content, flows, roles, data, and features from the website prompt, but adapts the UI/UX specifically for mobile with a more app-native experience. Replit Agent is designed for natural-language app building, so this prompt is written to be explicit, modular, and build-ready.

Prompt for Replit Agent — UTA FurniFind Mobile App
You are an expert mobile product designer, full-stack app architect, and ecommerce UX specialist.
Build a premium, high-definition, mobile-first application named UTA FurniFind for the University of Texas at Arlington community and Arlington donors.

This is the mobile app version of the UTA FurniFind platform.
It must preserve all functionality, content, user roles, pages, inventory structure, AI features, and 100 dummy records from the web version, but the mobile UI should be redesigned for phones with a more native app feel instead of a desktop website layout.

The app helps UTA students discover free furniture and household items currently available at Mission Arlington, so they do not need to visit early morning and wait in line just to check availability. It should function as a live mobile inventory and discovery app for community furniture and household goods.

This is not a normal ecommerce app. It is a community reuse marketplace / donation discovery mobile app with:

premium UI,

role-based access,

fast item discovery,

donation workflows,

student browsing,

community participation,

admin inventory management,

and strong visual storytelling through modern mobile animations and VFX.

1. Core product vision
Create a mobile platform where:

Students can browse available free items before visiting Mission Arlington.

Donors can submit items they want to donate.

Mission Arlington staff can manage inventory, approve listings, and update availability.

Community users can see impact, donation guidelines, and pickup information.

The app must include:

modern inventory browsing,

category filters,

search and sorting,

item detail pages,

save/bookmark/favorite features,

availability status,

pickup instructions,

donation submission flow,

community impact section,

responsive mobile-first design.

The experience should feel like a premium mobile marketplace + university service app.

Design the app specifically for mobile screens, not a desktop browser layout.

Mobile visual identity:
The app should feel like a blend of:

3. Best-fit mobile tech stack
Use the best modern stack for a production-ready mobile app:

Framework: React Native with Expo.

Navigation: Expo Router.

Styling: NativeWind, Tailwind-like styling, or a strong React Native design system.

Animation: Reanimated + Gesture Handler + Framer Motion style equivalents for mobile transitions.

State management: Zustand, Redux Toolkit, or Context.

Backend: Replit backend, Node.js/Express, or API routes.

Database: PostgreSQL or MongoDB.

Auth: Clerk, NextAuth-style auth if supported, or JWT-based role auth.

Image handling: optimized image components and local placeholder assets.

Charts / analytics: mobile-friendly chart library for inventory and community stats.

AI enhancements: optional AI helper for recommendations, donation guidance, and smart search.

If Replit Agent asks for a mobile starter, choose an Expo-based structure.
